# ブログ管理ガイド - 重要

## ⚠️ 必ず守るべきルール

### 新しいブログ記事を作成したら、必ず以下の3つのファイルを更新してください

1. **blog-data.json** - ブログデータベース（最重要）
2. **src/index.html** - トップページのBLOG_DATAセクション
3. **src/blog.html** - ブログ一覧ページのBLOG_DATAセクション

**これらのファイルが同期されていないと、ブログ一覧に記事が表示されません！**

---

## 📋 新規ブログ記事作成の完全チェックリスト

### ステップ1: HTMLファイルの作成
- [ ] `src/blog/記事名.html` を作成
- [ ] メタデータ（title, description, OGP, 構造化データ）を設定
- [ ] 画像パスを正しく設定
- [ ] 日付を正しく設定

### ステップ2: blog-data.jsonへの登録（必須）
- [ ] `blog-data.json`を開く
- [ ] 新しい記事のエントリを**日付順**に追加
- [ ] 以下の情報を正確に入力:
  - `id`: ファイル名（拡張子なし）
  - `title`: 記事タイトル
  - `date`: 公開日（YYYY-MM-DD形式）
  - `excerpt`: 記事の要約
  - `image`: 画像パス
  - `category`: カテゴリ
  - `url`: `blog/ファイル名.html`

### ステップ3: index.htmlの更新（必須）
- [ ] `src/index.html`を開く
- [ ] `window.BLOG_DATA`セクションを見つける
- [ ] blog-data.jsonと同じ内容を**日付順**に追加

### ステップ4: blog.htmlの更新（必須）
- [ ] `src/blog.html`を開く
- [ ] `window.BLOG_DATA`セクションを見つける
- [ ] blog-data.jsonと同じ内容を**日付順**に追加

### ステップ5: 画像ファイルの配置
- [ ] 使用する画像を`src/images/`または`public/images/`に配置
- [ ] 画像パスがHTMLとJSONで一致していることを確認

### ステップ6: 同期チェック（重要）
- [ ] 同期チェックスクリプトを実行: `./scripts/check-blog-data-sync.sh`
- [ ] エラーがないことを確認

### ステップ7: ローカルプレビュー
- [ ] ローカルサーバーで確認
- [ ] ブログ一覧に新しい記事が表示されることを確認
- [ ] 記事ページが正しく表示されることを確認

### ステップ8: アップロード準備
- [ ] 更新したファイルをアップロード用フォルダーにコピー
- [ ] アップロードチェックリストを確認

---

## 🛠️ 便利なスクリプト

### 1. 同期チェックスクリプト
```bash
./scripts/check-blog-data-sync.sh
```
**用途**: public/blog/内の全HTMLファイルがblog-data.jsonに登録されているか確認

### 2. メタデータ抽出スクリプト
```bash
./scripts/extract-blog-metadata.sh public/blog/記事名.html
```
**用途**: 既存のHTMLファイルからメタデータを抽出してJSON形式で出力

## 📝 blog-data.jsonのエントリ例

```json
{
  "id": "example-blog-post",
  "title": "記事のタイトル",
  "date": "2025-11-27",
  "excerpt": "記事の要約文。SEOに重要なので丁寧に書く。",
  "image": "/images/example.jpg",
```

## ⚠️ よくある間違い

### ❌ 間違い1: HTMLファイルだけ作成してblog-data.jsonを更新しない
**結果**: ブログ一覧に表示されない

### ❌ 間違い2: blog-data.jsonだけ更新してindex.htmlとblog.htmlを更新しない
**結果**: ブログ一覧に表示されない（BLOG_DATAがインライン化されているため）

### ❌ 間違い3: 日付順に並べない
**結果**: 新しい記事が一覧の下の方に表示される

### ❌ 間違い4: 画像パスが間違っている
**結果**: 画像が表示されない

### ❌ 間違い5: URLのパスが間違っている
**結果**: リンクが404エラーになる

## 🔍 トラブルシューティング

### Q: 新しい記事がブログ一覧に表示されない
**A**: 以下を確認してください
1. blog-data.jsonに記事が登録されているか
2. src/index.htmlのBLOG_DATAに記事が追加されているか
3. src/blog.htmlのBLOG_DATAに記事が追加されているか
4. 日付順に正しく並んでいるか
5. ブラウザのキャッシュをクリアしたか

### Q: 画像が表示されない
**A**: 以下を確認してください
1. 画像ファイルが正しい場所に配置されているか
2. 画像パスが正しいか（/images/で始まるか）
3. ファイル名の大文字小文字が一致しているか

### Q: リンクをクリックすると404エラーになる
**A**: 以下を確認してください
1. HTMLファイルが正しい場所に配置されているか
2. URLパスが正しいか（blog/ファイル名.html）
3. ファイル名が一致しているか

## 🎯 重要な注意事項

1. **3つのファイル（blog-data.json, index.html, blog.html）は常に同期させる**
2. **新規記事は必ず日付順（新しい順）に追加する**
3. **アップロード前に必ず同期チェックスクリプトを実行する**
4. **画像パスとURLパスは正確に記述する**
5. **カテゴリは既存のものから選ぶ（新規カテゴリは慎重に）**

---

## 📞 問題が発生したら

1. まず`./scripts/check-blog-data-sync.sh`を実行
2. エラーメッセージを確認
3. このガイドのトラブルシューティングを参照
4. それでも解決しない場合は、最新のバックアップから復元
